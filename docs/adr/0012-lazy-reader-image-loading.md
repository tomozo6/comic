# ADR 0012: リーダーのページ画像は表示範囲の前後だけ読み込む

- 状態: 採用
- 日付: 2026-07-20

## 背景

リーダーは表示範囲に近い画像だけを遅延読込する。しかし巻 API が全ページ分の GCS 署名 URL を
逐次発行してから応答するため、215 ページの巻では最初の画像を読み込み始める前の待ち時間が
長い。不要な署名発行と API 応答も通信量を増やす。

## 決定

- `IntersectionObserver` を使い、ビューポートに近いページだけへ画像 URL を設定する。
- 先読み範囲は、表示中ページの前後 3 画面分とする。
- 未読込ページは `min-height: 100vh` を確保したプレースホルダーとして表示し、画像本体は
  取得しない。画像寸法のカタログ化は必要になった場合に検討する。
- 巻 API は総ページ数と最初の 8 ページ分の URL だけを返す。追加の URL は
  `GET /api/manga/{mangaId}/volumes/{volumeId}/pages?start={start}` から 8 ページ単位で取得する。
- 各 URL バッチの署名は最大 8 件並列で発行し、応答内のページ順は維持する。
- リーダーはバッチ末尾の 3 ページ手前が先読み範囲に入ったとき、次のバッチを先行取得する。
- 一度読み込んだページは、その巻を開いている間は解放しない。逆方向へのスクロールで再取得を
  発生させず、通信量を増やさない。
- 署名 URL の有効期限切れや画像取得失敗時は、該当する URL バッチだけを再取得して一度だけ
  再試行する。

## 影響

- 巻を途中で閉じた場合、その後のページ画像と署名 URL は取得・発行されない。
- 初期表示で必要な IAM 署名要求は 215 件から最大 8 件になる。
- 高速スクロール時は、先読み範囲外の画像が短時間プレースホルダーになることがある。
- 長大な巻でメモリ問題が確認された場合だけ、遠いページを解放する方式を別途検討する。
