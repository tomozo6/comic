# ADR 0010: 実際の漫画画像は非公開 GCS バケットに保存する

- 状態: 採用
- 日付: 2026-07-19

## 背景

実際の漫画画像はリポジトリや GitHub Pages に置けない。閲覧は家族に限定し、画像を
公開 URL で配信せず、アプリケーションが発行する短命の URL だけで表示する必要がある。

## 決定

- GCP プロジェクト `tomozo6` に、東京リージョン（`asia-northeast1`）の GCS バケット
  `tomozo-manga-images` を作成する。
- バケットは非公開とし、Public Access Prevention と Uniform bucket-level access を有効に
  する。オブジェクト ACL ではアクセスを管理しない。
- 誤った上書きから復旧できるよう、オブジェクト バージョニングを有効にする。
- 初期の投入は管理者が `gcloud storage cp` などで手動実行する。ブラウザからの
  アップロード機能は作らない。
- アプリケーションの GCS 署名付き URL 発行と、実行サービス用 IAM は画像配信の実装時に
  追加する。バケットを公開する代替策は採らない。

## 影響

- Terraform 適用後、管理者はカタログの画像キーに対応するオブジェクトをバケットへ投入する。
- バケット名は全世界で一意である必要があり、名前の変更はデータ移行を伴う。
- バージョン保存には追加のストレージ料金がかかる。不要になった世代の整理手順は、運用量を
  確認してから別途定める。
