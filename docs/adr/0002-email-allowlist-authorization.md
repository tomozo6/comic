# ADR 0002: 閲覧認可にメールアドレス許可リストを使う

- 状態: 採用
- 日付: 2026-07-18

## 背景

Google を Firebase Authentication のログイン手段にすると、Google アカウントを
持つ人がログインを試みられる。サイトは家族限定であり、認証済みであることだけでは
閲覧を許可する条件として不十分である。

## 決定

- Go API は Firebase ID トークンを検証した後、トークン内のメールアドレスを
  サーバー設定の許可リストと照合する。
- 許可リストにない利用者には、作品・巻・画像を返さず `403 Forbidden` を返す。
- 許可リストは環境変数から読み取り、リポジトリには実際のメールアドレスを含めない。
- フロントエンドのログイン成功だけを閲覧許可の根拠にしない。各保護 API で
  認証と認可を行う。

## 理由

家族の人数が少ない初期段階では、運用とレビューが簡単で、誰にアクセスを許可して
いるかが明確になる。将来、より柔軟な管理が必要になれば Firebase Custom Claims 等へ
移行できる。

## 影響

- ローカル起動時にも許可するテスト用 Google アカウントを環境変数へ設定する。
- API のテストでは、未認証、許可済み、許可リスト外を区別して確認する。
