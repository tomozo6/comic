# ADR 0001: ローカル開発では短命なローカル配信 URL を発行する

- 状態: 廃止（[ADR 0013](0013-gcs-only-image-source.md) により置換）
- 日付: 2026-07-18

## 背景

本サイトは家族限定で、ログインには実際の Firebase Authentication を使う。
将来的な漫画画像の保管先は private GCS bucket とし、ブラウザには短命の
GCS 署名付き URL を渡す。ローカル開発でも、画像 URL の期限切れと再取得を
含むリーダーの挙動を確認したい。

## 決定

- 初期開発用の作品・ページには、公開可能なダミー画像だけを使う。実際の漫画
  画像はリポジトリに追加しない。
- ローカルでは Go アプリケーションが、認証済みの巻 API に対して短命な
  ローカル配信 URL を返す。
- ローカル配信 URL はアプリケーションがトークンの有効期限と対象画像を検証して
  から画像を返す。GCS V4 署名 URL を再現するものではない。
- ブラウザは配信先に依存せず、巻 API が返す一時 URL を表示する。URL の期限切れ時は
  巻 API を再取得する。
- 本番では同じ API 契約のまま、ローカル配信 URL を GCS 署名付き URL に置き換える。

## 理由

Firebase Storage Emulator はオブジェクト操作の開発には使えるが、GCS の V4 署名 URL
の完全な代替ではない。アプリケーション境界で URL 発行を抽象化すると、ローカルで
必要な閲覧体験を検証しつつ、本番への切替範囲を小さくできる。

## 影響

- API は画像オブジェクトキーではなく、表示用の一時 URL を返す。
- 実装ではローカル用 URL 発行器と GCS 用署名 URL 発行器を差し替え可能にする。
- ダミー画像はライセンス・出所を記録し、公開してよいものだけを使用する。
