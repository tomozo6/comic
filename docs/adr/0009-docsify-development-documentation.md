# ADR 0009: 開発・運用の設計書をDocsifyで公開する

- 状態: 採用
- 日付: 2026-07-19

## 背景

プロダクト仕様、アーキテクチャ、運用手順、意思決定をリポジトリ内で一緒に保守し、開発・運用者が
横断して読めるようにする必要がある。

## 決定

- `docs/` を開発・運用者向けの設計書の正本とし、Docsifyで表示する。
- GitHub Pagesは `docs/` を公開元にする。
- `_sidebar.md` をDocsifyの共通ナビゲーションとして使う。
- `.nojekyll` を `docs/` に置き、GitHub Pagesが `_sidebar.md` を公開対象から除外しないようにする。
- 文書の分類とDocsify設定は [開発ドキュメントの運用](../operations/documentation.md) に記録する。

## 影響

- 文書の追加・移動時は、ホームとサイドバーの両方を更新する。
- GitHub Pagesで公開されるため、秘密情報や個人情報を含めない。
