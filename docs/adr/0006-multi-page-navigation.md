# ADR 0006: Firebase Auth を使う複数ページ構成にする

- 状態: 採用
- 日付: 2026-07-18

## 背景

閲覧者が作品・巻・リーダーを移動したとき、URL の変化、ブラウザの戻る/進む、
リロード、特定の巻への直接アクセスを自然に扱いたい。単一 HTML 内で画面を
切り替える SPA はこの要件に適さない。

## 決定

- Go は次の HTML ページを個別の GET ルートで配信する。
  - `/` — ログイン
  - `/library` — 作品一覧
  - `/manga/{mangaId}` — 巻一覧
  - `/manga/{mangaId}/volumes/{volumeId}` — 漫画リーダー
- ページ間は通常のリンクで移動し、ブラウザが HTML を再読み込みする。
- 各保護ページは Firebase Auth のログイン状態を確認し、未ログインなら `/` へ移動する。
- 認証・メール許可リストの最終判定は引き続き Go API が行う。HTML の表示だけを
  アクセス許可の根拠にしない。

## 影響

- ページごとに必要な API を取得する JavaScript を持つ。
- URL をブックマーク・共有・再読み込みしても、対象の作品または巻を開ける。
- 画像 URL の期限切れ時は、現在の巻 API だけを再取得して復帰する。
