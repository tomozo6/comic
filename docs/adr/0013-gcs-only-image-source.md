# ADR 0013: 漫画画像はすべての環境で GCS から読む

- 状態: 採用
- 日付: 2026-07-22

## 背景

ローカル配信用の埋込み SVG、HMAC トークン、配信エンドポイント、画像ソースの切替設定を
維持すると、実際に読む GCS 経路とは別の実装とテストが必要になる。GCS にはヒストリエの
画像が配置済みであり、開発・テストでも実経路を使える。

## 決定

- 漫画画像の保存先は非公開 GCS バケット `tomozo-manga-images` だけとする。
- アプリケーションは固定の `manga-media-signer@tomozo6.iam.gserviceaccount.com` を使って
  GCS 署名 URL を発行する。ローカル画像、ローカル配信 URL、画像発行器の切替は持たない。
- ローカル開発と通常のテストは ADC で GCS のヒストリエへ実アクセスする。
- Cloud Run でも同じ ADC API を使う。デプロイ時には、Cloud Run 実行サービスアカウントへ
  `manga-media-signer` の Token Creator 権限を付与する。

## 影響

- 開発・テストにはネットワーク、ADC、GCS 権限が必須になる。
- GCS 画像を使えない環境ではアプリケーションと画像関連テストは実行できない。
- ローカル画像を保守する必要がなくなり、本番と同じ画像配信経路を検証できる。
