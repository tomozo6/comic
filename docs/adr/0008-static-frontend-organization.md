# ADR 0008: 実行用フロントエンドと設計資料を分離する

- 状態: 採用
- 日付: 2026-07-19

## 背景

フロントエンドのHTML、CSS、JavaScriptはアプリケーションが実際に配信する資産である。
一方、ワイヤーフレームは画面設計を共有・レビューするための資料であり、アプリケーションの
実行には不要である。

## 決定

- `application/public/` には実行時に配信する画面と、そのCSS・JavaScriptだけを置く。
- ワイヤーフレームなどの設計資料は `docs/design/` に置き、Docsifyのナビゲーションから参照する。
- 実行用のHTMLは構造と画面固有のスクリプト読み込みだけを持つ。共有処理はネイティブES Modules
  に分け、ビルドツールやフレームワークは導入しない。

## 影響

- `application/public/wireframe.html` は削除する。設計資料の正本は
  `docs/design/wireframes/v1.html` とする。
- 実行用画面の可読性を保つため、認証、API呼び出し、画面ごとの描画を別モジュールに分離する。
